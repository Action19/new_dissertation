# KIRISH

## 1. Tadqiqotning dolzarbligi va zarurati

XXI asr axborot jamiyati sharoitida ta'lim tizimini modernizatsiyalash, zamonaviy axborot-kommunikatsiya texnologiyalari (AKT) vositalaridan samarali foydalanish, ta'lim sifatini oshirish jahon hamjamiyati oldida turgan dolzarb masalalardan biridir[^1]. O'zbekiston Respublikasida amalga oshirilayotgan keng ko'lamli islohotlar, xususan, "Ta'lim to'g'risida"gi Qonun[^2], "Xalq ta'limi tizimini 2030-yilgacha rivojlantirish konsepsiyasi"[^3] hamda bir qator davlat dasturlari ta'lim sohasida raqamli texnologiyalar asosida o'quvchilar bilimini baholash tizimini tubdan takomillashtirishni taqozo etmoqda.

Hozirgi kunda jahon ta'lim amaliyotida o'quvchilar bilimini baholashning an'anaviy usullari (og'zaki so'rov, yozma nazorat ishi, test) o'z samaradorligini yo'qotib bormoqda. Buning asosiy sababi shundaki, zamonaviy ta'lim standartlari o'quvchilardan nafaqat bilimlarni eslab qolishni, balki ularni amalda qo'llash, tahlil qilish, sintez qilish va baholash kabi yuqori darajadagi kognitiv ko'nikmalarni egallashni talab qiladi[^4]. Shu munosabat bilan axborot texnologiyalari asosida bilimni baholash texnologiyalarini takomillashtirish zarurati tug'ilmoqda.

Informatika fani 9-10 sinflarda o'quvchilarning raqamli savodxonligi va dasturlash kompetensiyalarini shakllantirishda muhim o'rin tutadi. Biroq, mavjud baholash tizimlari ko'pincha nazariy bilimlarni tekshirishga qaratilgan bo'lib, amaliy ko'nikmalar, algoritmik fikrlash va muammoli vaziyatlarni yechish qobiliyatini to'liq baholay olmaydi[^5]. Bu esa o'qituvchilarning xolis va obyektiv baholash imkoniyatini cheklaydi, o'quvchilarning esa o'z rivojlanish dinamikasini tushunishiga to'sqinlik qiladi.

Mamlakatimizda 2019-2024 yillarda amalga oshirilgan ta'lim islohotlari natijasida umumiy o'rta ta'lim maktablarida informatika fanini o'qitish mazmuni tubdan qayta ko'rib chiqildi. Yangi Davlat ta'lim standarti (DTS) va o'quv dasturlari o'quvchilarda algoritmik fikrlash, dasturlash asoslari, ma'lumotlarni qayta ishlash va axborot xavfsizligi bo'yicha kompetensiyalar shakllantirishga yo'naltirildi[^6]. Shu bilan birga, ushbu kompetensiyalarni samarali baholash mexanizmlari etarli darajada ishlab chiqilmagan.

Xorijiy tajriba tahlili shuni ko'rsatadiki, rivojlangan mamlakatlarda (AQSH, Buyuk Britaniya, Finlandiya, Janubiy Koreya) axborot texnologiyalari asosida bilimni baholash tizimlari keng joriy etilgan va ular o'quvchilarning individual ehtiyojlariga moslashtirilgan, tezkor qayta aloqa (feedback) berishni ta'minlaydigan, turli xil topshiriq turlarini (test, loyiha, amaliy mashq, video-javob) qo'llab-quvvatlaydigan platformalar asosida tashkil etilgan[^7]. Ushbu tajribalarni mahalliy sharoitlarga moslashtirilgan holda joriy etish zaruriyati mavjud.



Tadqiqot dolzarbligini quyidagi omillar belgilaydi:

**Ijtimoiy-pedagogik jihat.** O'quvchilarning XXI asr kompetensiyalariga ega bo'lishi, raqamli iqtisodiyot talablariga javob beradigan kadrlar tayyorlash davlat ta'lim siyosatining ustuvor yo'nalishi hisoblanadi. Buning uchun o'quvchilar bilimini zamonaviy, xolis va ob'yektiv baholash tizimlari zarur.

**Ilmiy-nazariy jihat.** Pedagogika, didaktika va informatika metodikasi fanlarida bilimni baholashning nazariy asoslari mavjud, biroq axborot texnologiyalari asosida baholash texnologiyalarini takomillashtirish bo'yicha yetarlicha ilmiy-uslubiy tadqiqotlar olib borilmagan.

**Amaliy-metodologik jihat.** Informatika fanidan o'quvchilar bilimini baholashda zamonaviy dasturiy vositalar, adaptiv testlar, real vaqt rejimida qayta aloqa berish mexanizmlari, o'quvchi yutuqlarini kuzatish va tahlil qilish platformalari etarli darajada qo'llanilmaydi.

**Texnologik jihat.** Sun'iy intellekt, mashinali o'rganish, katta ma'lumotlarni tahlil qilish (Big Data), bulutli texnologiyalar pedagogik jarayonlarda, jumladan, bilimni baholashda qo'llanilish istiqbollariga ega, ammo ushbu yo'nalishda nazariy va amaliy izlanishlar yetarli emas.

Tadqiqot mavzusining respublika fan va texnologiyalar rivojlanishining ustuvor yo'nalishlariga mosligi O'zbekiston Respublikasi Prezidentining 2017-yil 7-fevraldagi PF-4947-son "O'zbekiston Respublikasini yanada rivojlantirish bo'yicha Harakatlar strategiyasi to'g'risida"gi, 2019-yil 29-apreldagi PF-5712-son "Xalq ta'limi tizimini 2030-yilgacha rivojlantirish konsepsiyasini tasdiqlash to'g'risida"gi, 2020-yil 30-apreldagi PF-5992-son "Raqamli iqtisodiyot va elektron hukumatni rivojlantirish chora-tadbirlari to'g'risida"gi Farmonlari hamda O'zbekiston Respublikasi Vazirlar Mahkamasining 2020-yil 24-aprel 257-son "Umumta'lim maktablarida raqamli texnologiyalarni joriy etish chora-tadbirlari to'g'risida"gi qarori ijrosi bilan bevosita bog'liq[^8].

## 3. Tadqiqotning maqsadi

Tadqiqotning asosiy maqsadi 9-10 sinf informatika darslarida axborot texnologiyalari asosida o'quvchilar bilimini samarali baholash texnologiyasini ishlab chiqish va amaliyotga joriy etishdan iborat.

## 4. Tadqiqotning vazifalari

Tadqiqot maqsadiga erishish uchun quyidagi vazifalar belgilandi:

1. Yuqori sinf o'quvchilarni axborot texnologiyalari asosida bilimini baholashning ilmiy-nazariy asoslarini tahlil qilish va muammoning pedagogik mohi yatini aniqlash;

2. O'quvchilar bilimini baholashning xorijiy ilg'or tajribalari va mahalliy tadqiqotlarni qiyosiy tahlil qilib, O'zbekiston ta'lim tizimi uchun qo'llash imkoniyatlarini asoslash;

3. Yuqori sinf informatika darslarida axborot texnologiyalari asosida bilimni baholashning didaktik modelini ishlab chiqish;

4. Baholash mezonlari, ko'rsatkichlari va darajalarini ilmiy asoslash hamda ularni amaliyotda qo'llash uslubiyotini yaratish;

5. Axborot texnologiyalari asosida bilimni baholash platformasini ishlab chiqish va uning funksional imkoniyatlarini aniqlash;

6. Ishlab chiqilgan texnologiyaning samaradorligini pedagogik tajriba-sinov ishlari orqali tekshirish va statistik tahlil qilish.



## 5. Tadqiqotning obyekti va predmeti

**Tadqiqotning obyekti:** 9-10 sinf o'quvchilarining informatika fanidan bilim, ko'nikma va malakalarini baholash jarayoni.

**Tadqiqotning predmeti:** 9-10 sinf informatika darslarida axborot texnologiyalari asosida o'quvchilar bilimini baholash texnologiyasi, uning tarkibiy qismlari, didaktik modeli va amaliy joriy etish mexanizmlari.

## 6. Tadqiqotning ilmiy farazi (gipotezasi)

Agar 9-10 sinf informatika darslarida axborot texnologiyalari asosida o'quvchilar bilimini baholash tizimi quyidagi pedagogik shartlar asosida tashkil etilsa:

- Bloom taksonomiyasiga asoslangan ko'p darajali baholash mezonlari qo'llanilsa;
- Adaptiv test usullari va turli xil topshiriq turlari (test, ochiq javob, amaliy loyiha, video-taqdimot) integratsiyalashtirilsa;
- Real vaqt rejimida qayta aloqa (feedback) berish va individual ta'lim traektoriyasini aniqlash mexanizmlari ishlatilsa;
- O'qituvchi va o'quvchi uchun vizual-analitik hisobotlar (dashboard) taqdim etilsa;
- O'zbek tili interfeysi va milliy ta'lim standartlari talablariga moslashtirilgan bo'lsa,

unda o'quvchilarning bilim sifati, algoritmik fikrlash va amaliy dasturlash ko'nikmalari sezilarli darajada oshadi, baholash jarayoni obyektiv, shaffof va samarali bo'ladi.

## 7. Tadqiqotning usullari

Tadqiqot maqsad va vazifalariga ko'ra quyidagi usullar qo'llanildi:

**Nazariy usullar:** ilmiy-pedagogik, psixologik, didaktik adabiyotlarni tahlil qilish va sintez qilish; qonun-qoidalar va me'yoriy hujjatlarni o'rganish; xorijiy va mahalliy tajribani qiyosiy tahlil qilish; pedagogik modellashtirish; tizimli yondashuv va tizimli tahlil.

**Empirik usullar:** pedagogik kuzatish; o'qituvchilar va o'quvchilar bilan so'rovnoma, intervyu; ekspert baholash; pedagogik eksperiment (konstatlovchi, shakllantiruvchi, nazorat bosqichlari); o'quvchilar bilimini test orqali o'lchash; o'quv jarayonini monitoring qilish.

**Matematik-statistik usullar:** ma'lumotlarni miqdoriy va sifat jihati tahlil qilish; natijalarni grafik va jadval ko'rinishida aks ettirish; Styudent t-mezoni va xи-kvadrat mezoni orqali statistik ahamiyatlilikni aniqlash; dispersi analiz; korrelyatsion tahlil.

**Dasturlash va texnologik usullar:** veb-platforma ishlab chiqish (React.js, Node.js, MongoDB); responsiv dizayn (Tailwind CSS); RESTful API yaratish; sun'iy intellekt kutubxonalarini (TensorFlow.js) integratsiyalash; bulutli xizmatlardan foydalanish.



## 8. Tadqiqotning ilmiy yangiligi

Tadqiqotning ilmiy yangiligi quyidagilardan iborat:

**1.** Informatika fanidan 9-10 sinf o'quvchilarini baholashning didaktik modeli ishlab chiqilgan bo'lib, unda maqsad-muammoli, tarkibiy-funksional va natija-baholash komponentlari hamda ularning o'zaro bog'liqligi asoslab berilgan.

**2.** O'quvchilar bilimini axborot texnologiyalari asosida baholashning ko'p darajali mezonlar tizimi (bilish, tushunish, qo'llash, tahlil qilish, sintez, baholash) va har bir daraja uchun indikatorlar to'plami Bloom taksonomiyasi va milliy ta'lim standartlari asosida takomillashtirilgan.

**3.** Adaptiv baholash algoritmlari, turli shakldagi topshiriqlarni (test, ochiq javob, kod yozish, loyiha baholash) avtomatlashtirishning ilmiy-pedagogik asoslari va texnik imkoniyatlari aniqlangan.

**4.** Sun'iy intellekt yordamida avtomatik topshiriq generatsiyasi, javoblarni baholash va o'quvchiga individual qayta aloqa berish mexanizmlarining pedagogik modeli ishlab chiqilgan va amaliyotga tatbiq etilgan.

**5.** O'qituvchi va o'quvchi uchun vizual-analitik boshqaruv paneli (dashboard) orqali bilim dinamikasini kuzatish, zaif tomonlarni aniqlash va individual ta'lim yo'llarini loyihalash uslubiyoti yaratilgan.

# I BOB. YUQORI SINF O'QUVCHILARNI AXBOROT TEXNOLOGIYALARI ASOSIDA BILIMINI BAHOLASHNING ILMIY-NAZARIY ASOSLARI

## 1.1. Yuqori sinf o'quvchilarni axborot texnologiyalari asosida bilimini baholash – pedagogik muammo sifatida

Zamonaviy ta'lim tizimida o'quvchilar bilimini baholash masalasi doimo dolzarb bo'lib kelgan va pedagogik jarayonning muhim tarkibiy qismi hisoblanadi. Baholash nafaqat o'quvchining bilim darajasini aniqlashga, balki ta'lim jarayonini takomillashtirish, o'qituvchining faoliyatini ob'yektiv baholash va ta'lim sifatini oshirishga xizmat qiladi[^16].

Pedagogik adabiyotlarda "baholash" ("assessment") va "baho qo'yish" ("grading") tushunchalari farqlanadi. Baholash – bu o'quvchining bilim, ko'nikma va kompetensiyalari to'g'risida ma'lumot to'plash va tahlil qilish jarayoni, baho qo'yish esa ushbu ma'lumotlarni belgilangan shkala asosida ifodalash demakdir[^17]. Zamonaviy yondashuvga ko'ra, baholash formativ (shakllantiruvchi) va summativ (yakunlovchi) turlarga bo'linadi.

**Formativ baholash** o'quv jarayoni davomida amalga oshirilib, o'quvchining rivojlanishini kuzatish, zaif tomonlarni aniqlash va darhol tuzatish imkonini beradi. Bu turdagi baholash o'quvchiga tezkor qayta aloqa (feedback) berishga qaratilgan va o'quvchining o'z-o'zini baholash qobiliyatini rivojlantirishga yordam beradi[^18].

**Summativ baholash** ma'lum bir davr (chorak, yarim yillik, yil) oxirida o'quvchining erishgan natijalarin jami baholashga mo'ljallangan. Bu turdagi baholash ko'proq nazorat va hisobot funksiyasini bajaradi, ammo o'quvchining keyingi rivojlanishiga to'g'ridan-to'g'ri ta'sir etmaydi[^19].

XXI asrda ta'lim paradigmasi o'zgarishi bilan an'anaviy baholash usullari (5 ballik shkala, og'zaki so'rov, yozma nazorat ishi) o'z samaradorligini yo'qotib bormoqda. Buning sabablari quyidagilar:

1. **Subyektivlik.** O'qituvchining shaxsiy qarashi, kayfiyati, o'quvchiga nisbatan munosabati bahoga ta'sir ko'rsatishi mumkin;

2. **Bir tomonlamalik.** Asosan nazariy bilimlar tekshiriladi, amaliy ko'nikmalar va kompetensiyalar yetarlicha baholanmaydi;

3. **Kechikkan qayta aloqa.** O'quvchi o'z xatolarini bir necha kun yoki hafta o'tgach bilib oladi, bu esa samarali o'rganishga to'sqinlik qiladi;

4. **Individual yondashuvning yo'qligi.** Barcha o'quvchilarga bir xil murakkablikdagi topshiriqlar beriladi, bu esa qobiliyatli o'quvchilar uchun juda oson, zaif o'quvchilar uchun esa juda qiyin bo'lishi mumkin;

5. **Shaffoflik etishmasligi.** O'quvchi va ota-onalar baholash mezonlarini to'liq tushunishmaydi, natijada bahoga ishonch pasayadi[^20].



Axborot texnologiyalarining rivojlanishi ta'lim sohasida, xususan, bilimni baholashda yangi imkoniyatlarni ochib berdi. **Axborot texnologiyalari asosida baholash** (Technology-Enhanced Assessment – TEA) deganda quyidagi komponentlar tushuniladi:

- Kompyuterli testing (CBT – Computer-Based Testing);
- Onlayn platformalar orqali topshiriqlar berish va natijalarni avtomatik tekshirish;
- Adaptiv testlar (CAT – Computerized Adaptive Testing);
- Elektron portfolio;
- Gamifikatsiya elementlari (badges, leaderboards);
- Sun'iy intellekt yordamida baholash;
- Katta ma'lumotlarni tahlil qilish (Learning Analytics);
- Real vaqt rejimida qayta aloqa berish tizimlari[^21].

Axborot texnologiyalari asosida baholashning asosiy afzalliklari quyidagilardan iborat:

**1. Obyektivlik va ishonchlilik.** Kompyuter orqali baholashda insoniy omil ta'siri kamayadi, natijada baho ob'yektiv bo'ladi. Test natijalari avtomatik tarzda qayta ishlanadi va statistik tahlil qilinadi, bu esa baholashning ishonchlilikni oshiradi[^22].

**2. Tezkorlik va samaradorlik.** Elektron testlar bir vaqtning o'zida yuzlab o'quvchiga amalga oshirilishi mumkin, natijalar darhol chiqadi. O'qituvchi vaqtini tekshirish ishlariga emas, balki pedagogik tahlil va metodish ishlariga sarflaydi[^23].

**3. Individual yondashuv.** Adaptiv testlar o'quvchining bilim darajasiga qarab topshiriq murakkabligini o'zgartiradi. Bu har bir o'quvchini o'zining "proksimal rivojlanish zonasi" (L.S.Vigotskiy) doirasida baholash imkonini beradi[^24].

**4. Ko'p qirralik va turli shakldagi topshiriqlar.** Axborot texnologiyalari nafaqat test, balki ochiq javobli savollar, video-taqdimotlar, interaktiv simulyatsiyalar, loyihaviy ishlar, kod yozish kabi turli shakldagi topshiriqlarni qo'llab-quvvatlaydi[^25].

**5. Uzluksiz monitoring va tahlil.** Elektron tizimlar o'quvchining butun ta'lim davri mobaynida erishgan natijalarini to'playdi, dinamikasini kuzatadi va prognoz qilish imkonini beradi. Bu formativ baholashning samaradorligini oshiradi[^26].

**6. Shaffoflik va ochiqlik.** O'quvchi va ota-onalar istalgan vaqtda baholar, statistika, tahlillar bilan tanishishlari mumkin. Baholash mezonlari aniq va tushunarli bo'ladi[^27].

Biroq, axborot texnologiyalari asosida baholashning bir qator muammolari ham mavjud:

- Texnik infratuzilma etishmasligi (barcha maktablarda zamonaviy kompyuter xonalari, internet aloqasi mavjud emas);
- O'qituvchilarning raqamli kompetensiyalari yetarli emasligi;
- O'quvchilarning kompyuterdan foydalanish ko'nikmalari turlicha bo'lishi (bu test natijalariga ta'sir qilishi mumkin);
- Elektron baholash tizimlarini yaratish va saqlab turish qimmat;
- Axborot xavfsizligi va shaxsiy ma'lumotlarni himoya qilish masalalari;
- Test topshirish jarayonida nusxa ko'chirish (cheating) xavfi[^28].



Informatika fani kontekstida bilimni baholashning o'ziga xos xususiyatlari mavjud. Informatika – bu amaliy fan bo'lib, o'quvchilar nafaqat nazariy bilimlarni, balki dasturlash, algoritmlar tuzish, kompyuter tizimlaridan foydalanish ko'nikmalarini egallashlari kerak. Shuning uchun informatika fanidan baholash quyidagi komponentlarni qamrab olishi zarur[^29]:

1. **Nazariy bilimlar:** asosiy tushunchalar, ta'riflar, klassifikatsiyalar, qonuniyatlar;
2. **Algoritmik fikrlash:** muammoni tahlil qilish, algoritm tuzish, optimal yechim topish;
3. **Amaliy ko'nikmalar:** dasturlash tillari, dasturiy vositalardan foydalanish;
4. **Ijodiy kompetensiyalar:** loyiha ishlab chiqish, yangi yechimlar topish;
5. **Axborot madaniyati:** raqamli etiket, axborot xavfsizligi, intellektual mulk huquqlarini hurmat qilish.

9-10 sinflarda informatika fanining mazmuni chuqur va keng qamrovli bo'lib, quyidagi asosiy bo'limlarni o'z ichiga oladi[^30]:

- **Algoritmlashtirish va dasturlash asoslari** (Python, C++, JavaScript);
- **Ma'lumotlar tuzilmasi** (massivlar, ro'yxatlar, steklar, navbatlar, daraxtlar);
- **Kompyuter tarmoqlari va internet** (TCP/IP, domen tizimi, veb-texnologiyalar);
- **Axborot xavfsizligi** (shifrlash, autentifikatsiya, viruslar, xavfsiz internet);
- **Ma'lumotlar bazasi** (SQL, jadvallar bilan ishlash);
- **Multimediya va grafika** (rasm, video, audio ishlov berish);
- **Sun'iy intellekt asoslari** (mashinali o'rganish g'oyalari, neyron tarmoqlar haqida umumiy tushuncha).

Ushbu mavzularni baholash uchun turli xil topshiriq turlari kerak: testlar (bilimni tekshirish uchun), dasturlash mashqlari (kod yozish ko'nikmalarini baholash uchun), loyiha ishlari (ijodiy yondashuvni tekshirish uchun), taqdimotlar (kommunikativ kompetensiyalarni baholash uchun)[^31].

Xulosa qilib aytganda, yuqori sinf o'quvchilarni informatika fanidan axborot texnologiyalari asosida baholash – bu nafaqat texnik vosita, balki pedagogik yondashuvdir. U zamonaviy ta'lim tizimining talablariga javob berishi, ob'yektivlik, shaffoflik, individual yondashuvni ta'minlashi, shu bilan birga, pedagogik maqsadga xizmat qilishi – o'quvchini rivojlantirishi va uning ta'lim motivatsiyasini oshirishi kerak.




## 1.2. Yuqori sinf o'quvchilarni axborot texnologiyalari asosida bilimini baholashning pedagogik-psixologik o'ziga xos xususiyatlari

15-17 yoshli o'quvchilar (9-10 sinflar) psixologik rivojlanishining o'ziga xos xususiyatlariga ega bo'lgan yoshdir. Bu – o'smirlik davri oxiri va erta yoshlik davri boshlanishi, shaxsning intensiv rivojlanishi, o'z-o'zini anglash, mustaqillik, tanqidiy fikrlash shakllanish davri[^32].

Pedagogik psixologiya nuqtai nazaridan ushbu yoshdagi o'quvchilarni baholashda quyidagi xususiyatlarni hisobga olish zarur:

**1. Motivatsiya va o'z-o'zini baholash qobiliyati.** Yuqori sinf o'quvchilarida tashqi motivatsiya (baho olish, ota-onani xursand qilish) asta-sekin ichki motivatsiyaga (bilim olishdan zavqlanish, o'z-o'zini rivojlantirish istagi) o'tadi. Shu sababli baholash tizimi nafaqat nazorat funksiyasini, balki o'quvchini rag'batlantirish va yo'naltirish funksiyasini ham bajarishi kerak[^33].

L.S.Vigotskiy ta'kidlaganidek, o'quvchining "proksimal rivojlanish zonasi" (zona blijayshego razvitiya) doirasida ishlash muhim ahamiyatga ega. Ya'ni, topshiriqlar juda oson (o'quvchi allaqachon biladi) yoki juda qiyin (o'quvchi umuman hal qila olmaydi) bo'lmasligi kerak. Optimal murakkablik darajasi o'quvchini qiziqtiradi va uning rivojlanishiga yordam beradi[^34].

**2. Tanqidiy fikrlash va mustaqillik.** 9-10 sinf o'quvchilari allaqachon tanqidiy fikrlash qobiliyatiga ega. Ular bahoning adolatli yoki adolatsiz ekanligini tahlil qilishlari, o'qituvchining qaroriga e'tiroz bildirishi mumkin. Shuning uchun baholash jarayoni shaffof, tushunarli va obyektiv bo'lishi kerak[^35].

**3. Raqobatbardoshlik va hamkorlik.** Ushbu yoshdagi o'quvchilarda raqobatchi ruh kuchli bo'ladi. Leaderboard, reytinglar kabi gamifikatsiya elementlari ularni qo'shimcha motivatsiya qilishi mumkin. Biroq, haddan tashqari raqobat salbiy oqibatlarga (stress, tashvish, do'stona munosabatlarning buzilishi) olib kelishi mumkin. Shuning uchun baholash tizimi ham individual rivojlanishni, ham jamoaviy ishlarni qo'llab-quvvatlashi kerak[^36].

**4. Texnologiyalarga moslashuvchanlik.** Zamonaviy o'quvchilar "raqamli avlod" (digital natives) hisoblanadi. Ular kompyuter, smartfon, planshetlardan bemalol foydalanadi, ijtimoiy tarmoqlarda faol[^37]. Axborot texnologiyalari asosida baholash ularga tanish va qulay muhit yaratadi. Biroq, ba'zi o'quvchilarda texnik ko'nikmalar yetarli bo'lmasligi mumkin, bu ham hisobga olinishi kerak.

**5. Tezkor qayta aloqa (feedback) zaruriyati.** Psixologik tadqiqotlar shuni ko'rsatadiki, baholash natijalari qanchalik tezroq ma'lum qilinsa, o'rganish samaradorligi shunchalik yuqori bo'ladi. An'anaviy tizimda o'quvchi nazorat ishini topshirgandan so'ng 1-2 hafta kutishi kerak edi. Axborot texnologiyalari esa darhol natija berishga imkon yaratadi[^38].

**6. Xatolardan o'rganish imkoniyati.** Carol Dweck ning "o'sish qarashlari" (growth mindset) nazariyasiga ko'ra, xatolar o'rganishning tabiiy qismi bo'lib, ulardan to'g'ri foydalanish orqali bilimni mustahkamlash mumkin. Axborot texnologiyalari asosida baholash tizimi o'quvchiga xatolarini tahlil qilish, qayta urinish, o'z rivojlanish dinamikasini kuzatish imkonini berishi kerak[^39].

**7. Differentsiatsiya va individual ta'lim yo'llari.** Har bir o'quvchining qobiliyati, bilim darajasi, o'rganish tezligi har xil. Adaptiv baholash tizimlari har bir o'quvchiga mos keladigan murakkablik darajasidagi topshiriqlarni taklif qiladi, bu esa samarali o'rganishni ta'minlaydi[^40].




Bloom taksonomiyasi (qayta ko'rib chiqilgan versiyasi) kognitiv jarayonlarni olti darajaga ajratadi[^41]:

1. **Eslab qolish (Remember)** – faktlar, ta'riflar, qoidalarni eslab qolish;
2. **Tushunish (Understand)** – ma'noni anglash, o'z so'zlari bilan izohlash;
3. **Qo'llash (Apply)** – bilimlarni yangi vaziyatlarda qo'llash;
4. **Tahlil qilish (Analyze)** – tuzilishni, qismlarni ajratish, munosabatlarni aniqlash;
5. **Baholash (Evaluate)** – tanqidiy baholash, qaror qabul qilish;
6. **Yaratish (Create)** – yangi narsalarni yaratish, sintez qilish.

Informatika fanidan o'quvchilarni baholashda ushbu barcha darajalarni qamrab olish muhim. Masalan:

- **Eslab qolish darajasi:** "Algoritm nima?" degan savolga javob berish;
- **Tushunish darajasi:** Berilgan algoritm nima qilishini o'z so'zlari bilan tushuntirish;
- **Qo'llash darajasi:** Berilgan muammoni hal qilish uchun mos algoritmni tanlash va qo'llash;
- **Tahlil qilish darajasi:** Ikki xil algoritmning samaradorligini taqqoslash;
- **Baholash darajasi:** Qaysi algoritm ma'lum vaziyat uchun yaxshiroq ekanligini asoslab berish;
- **Yaratish darajasi:** Yangi muammo uchun o'z algoritmingizni yaratish[^42].

Pedagogik tajribalar shuni ko'rsatadiki, ko'pchilik o'quvchilar faqat past darajadagi topshiriqlar (eslab qolish, tushunish) bilan shug'ullanadilar, yuqori darajadagi kognitiv ko'nikmalarga esa kam e'tibor beriladi. Axborot texnologiyalari asosida baholash tizimi barcha darajalarni muvozanatli tarzda qamrab olishi va o'quvchilarni yuqori darajadagi fikrlashga yo'naltirishi kerak[^43].

Xulosa qilib aytganda, yuqori sinf o'quvchilarini informatika fanidan axborot texnologiyalari asosida baholashda pedagogik-psixologik yondashuvlar asosiy o'rin tutadi. Texnologiya – bu vosita, asosiy maqsad esa o'quvchining har tomonlama rivojlanishi, uning bilim olishga bo'lgan qiziqishini oshirish va XXI asr kompetensiyalarini shakllantirishdir.

## 1.3. Xorijiy mamlakatlarda muammoning moxiyati va qiyosiy tahlil

Rivojlangan mamlakatlarda axborot texnologiyalari asosida o'quvchilar bilimini baholash tizimlarini joriy etishda katta tajriba to'plangan. Ushbu tajribalarni tahlil qilish O'zbekiston ta'lim tizimi uchun foydali xulosalar chiqarishga imkon beradi.

**AQSh tajribasi.** Amerika Qo'shma Shtatlarida kompyuterli testing (CBT) keng tarqalgan. SAT (Scholastic Assessment Test), GRE (Graduate Record Examination), TOEFL kabi milliy darajadagi imtihonlar butunlay kompyuter orqali amalga oshiriladi. Bundan tashqari, Ko'pgina davlat maktablarida PARCC (Partnership for Assessment of Readiness for College and Careers) va Smarter Balanced kabi standartlashtirilgan testlar qo'llaniladi[^44].

AQSHda keng qo'llaniladigan onlayn platformalar:

- **Khan Academy** – bepul ta'lim platformasi bo'lib, matematika, informatika, fanlar bo'yicha video darslar va interaktiv mashqlarni taklif qiladi. Har bir o'quvchining rivojlanishi kuzatiladi va individual ta'lim yo'li tavsiya etiladi[^45];
- **Google Classroom** – o'qituvchilarga topshiriqlar berish, natijalarni tekshirish, o'quvchilar bilan muloqot qilish imkonini beruvchi bepul platforma;
- **Edmodo** – ijtimoiy tarmoq elementlari bilan boyitilgan ta'lim platformasi;
- **Coursera, edX** – oliy ta'lim darajasidagi onlayn kurslar va sertifikatlar taklif qiladi.

Informatika ta'limida esa **Code.org, Scratch, Codecademy, LeetCode** kabi maxsus platformalar qo'llaniladi. Ular o'quvchilarga dasturlashni o'rganish, kod yozish, masalalar yechish va natijalarini avtomatik tekshirish imkonini beradi[^46].




**Buyuk Britaniya tajribasi.** Britaniyada ta'lim tizimi 2014-yilda yangilangan Milliy o'quv dasturida (National Curriculum) informatika faniga alohida e'tibor qaratilgan. 5 yoshdan boshlab barcha o'quvchilar dasturlash, algoritmlash, raqamli savodxonlik asoslarini o'rganadilar. GCSE (General Certificate of Secondary Education) va A-Level imtihonlari qisman kompyuter orqali topshiriladi[^47].

Britaniyada **BBC Bitesize** platformasi maktab o'quvchilari uchun barcha fanlar bo'yicha interaktiv darslar, testlar va o'yinlarni taklif qiladi. **Raspberry Pi Foundation** esa dasturlash va kompyuter fanlari bo'yicha bepul resurslar taqdim etadi[^48].

**Finlandiya tajribasi.** Finlandiya ta'lim tizimi jahonda eng yaxshilaridan biri hisoblanadi (PISA reytingi bo'yicha). Finlandiyada baholar 1-6 sinflarda umuman qo'yilmaydi, asosiy e'tibor formativ baholash va o'quvchiga individual yondashuv ga qaratilgan. 7-sinfdan boshlab raqamli baholash (4-10 shkala) qo'llanadi, ammo bu ham o'quvchining rivojlanishiga xizmat qiladi, nazorat emas[^49].

Finlandiyada har bir o'quvchiga shaxsiy ta'lim rejasi (Personal Learning Plan) tuziladi va u elektron tizimda saqlanadi. O'quvchi, ota-onalar va o'qituvchilar ushbu rejani muntazam ko'rib chiqadilar va yangilaydilar[^50].

**Janubiy Koreya tajribasi.** Janubiy Koreya texnologiyalar sohasida etakchi mamlakatlardan biri. 2011-yilda "Smart Education" milliy dasturi e'lon qilindi, uning maqsadi – barcha maktablarni raqamli texnologiyalar bilan ta'minlash, elektron darsliklar yaratish, onlayn ta'lim platformalarini joriy etish[^51].

Janubiy Koreyada **EBS (Educational Broadcasting System)** davlat ta'lim telekanali va onlayn platformasi barcha maktab o'quvchilari uchun bepul darslar, testlar va maslahatlar taqdim etadi. Informatika fanlari bo'yicha **SW Education** platformasi dasturlash, robotexnika, sun'iy intellekt bo'yicha ta'lim beradi[^52].

**Singapur tajribasi.** Singapur matematik va informatika ta'limida yuqori natijalarga erishgan (TIMSS va PISA reytinglari bo'yicha). Singapur hukumati 1997-yidan boshlab "Masterplan for IT in Education" dasturini amalga oshirmoqda. Hozirgi kunda barcha maktablar yuqori tezlikdagi internet, zamonaviy kompyuter xonalari, interaktiv doska va planshetlar bilan ta'minlangan[^53].

Singapurda **Student Learning Space (SLS)** milliy elektron ta'lim platformasi barcha fanlar bo'yicha onlayn darslar, interaktiv topshiriqlar, testlar va o'yinlar taklif qiladi. Platform adaptiv o'rganish texnologiyalariga asoslangan va har bir o'quvchining ehtiyojlariga moslashadi[^54].

**Estoniya tajribasi.** Estoniya – eng raqamlashgan davlatlardan biri. 2012-yilda "ProgeTiger" dasturi orqali barcha maktablarda dasturlash ta'limini joriy etish boshlandi. Hozirda 1-sinfdan boshlab o'quvchilar dasturlash, robotexnika, 3D dizayn bilan shug'ullanadi[^55].

Estoniyada **eKool** elektron jurnal va ta'lim boshqaruv tizimi keng qo'llaniladi. U orqali o'qituvchilar topshiriqlar beradi, baholar qo'yadi, ota-onalar esa real vaqt rejimida farzandlarining o'qish jarayonini kuzatadilar[^56].




**Qiyosiy tahlil va xulosalar.** Xorijiy tajribalar tahlili asosida quyidagi umumiy tendensiyalar va yaxshi amaliyotlarni ajratish mumkin:

1. **Davlat siyosati va investitsiyalar.** Barcha yuqorida tilga olingan mamlakatlarda ta'limni raqamlashtirish davlat darajasida qo'llab-quvvatlanadi va unga katta mablag' ajratiladi.

2. **Infrastruktura.** Maktablar zamonaviy texnik vositalar (kompyuter, planshet, internet) bilan to'liq ta'minlangan.

3. **O'qituvchilarni tayyorlash.** O'qituvchilar raqamli texnologiyalardan foydalanishga maxsus tayyorlanadi, muntazam malaka oshirish kurslari o'tkariladi.

4. **Milliy platformalar.** Ko'pchilik mamlakatlarda o'z milliy elektron ta'lim platformalari mavjud (Khan Academy, BBC Bitesize, EBS, SLS).

5. **Adaptiv o'rganish.** Zamonaviy platformalar sun'iy intellekt yordamida har bir o'quvchiga individual ta'lim yo'lini taklif qiladi.

6. **Bepul va ochiq resurslar.** Ko'pchilik ta'lim resurslari bepul va barcha uchun ochiq (open educational resources – OER).

7. **Formativ baholash ustuvorligi.** An'anaviy summativ baholash (yakuniy imtihon, baho) o'rnini formativ baholash (doimiy kuzatuv, tezkor qayta aloqa) egallayapti.

8. **Shaffoflik va ota-onalarning ishtiroki.** Elektron jurnal va platformalar orqali ota-onalar farzandlarining ta'lim jarayonida faol ishtirok etadilar[^57].

O'zbekiston ta'lim tizimi uchun ushbu tajribalarni to'g'ridan-to'g'ri nusxalash mumkin emas, chunki ijtimoiy-iqtisodiy sharoitlar, madaniy kontekst, texnik imkoniyatlar har xil. Biroq, quyidagi jihatlarni hisobga olgan holda moslashtirilgan yechimlar ishlab chiqish mumkin:

- Milliy ta'lim standartlari va o'quv dasturlariga muvofiqlik;
- O'zbek tilida interfeys va kontent;
- Mahalliy texnik imkoniyatlarga moslashtirish (past internet tezligi, eski kompyuterlar bilan ham ishlash);
- Iqtisodiy jihatdan maqbul (arzon yoki bepul) yechimlar;
- O'qituvchilarni bosqichma-bosqich tayyorlash va qo'llab-quvvatlash[^58].

## Birinchi bob bo'yicha xulosalar

1. Yuqori sinf o'quvchilarni informatika fanidan axborot texnologiyalari asosida bilimini baholash – bu zamonaviy ta'lim tizimining muhim tarkibiy qismi bo'lib, obyektivlik, shaffoflik, tezkorlik, individual yondashuvni ta'minlaydi.

2. An'anaviy baholash usullari (og'zaki so'rov, yozma nazorat ishi) subyektivlik, bir tomonlamalik, kechikkan qayta aloqa kabi kamchiliklarga ega, bu esa zamonaviy ta'lim talablariga javob bermaydi.

3. Axborot texnologiyalari asosida baholash kompyuterli testing, adaptiv testlar, elektron portfolio, gamifikatsiya, sun'iy intellekt yordamida baholash kabi turli shakllarni qamrab oladi.

4. Yuqori sinf o'quvchilari (15-17 yosh) psixologik jihatdan mustaqillik, tanqidiy fikrlash, o'z-o'zini baholash qobiliyati shakllanish davrida bo'lib, baholash tizimi ushbu xususiyatlarni hisobga olishi kerak.

5. Bloom taksonomiyasi kognitiv jarayonlarning oltita darajasini (eslab qolish, tushunish, qo'llash, tahlil qilish, baholash, yaratish) ajratadi va baholash tizimi barcha darajalarni qamrab olishi zarur.

6. Xorijiy mamlakatlarda (AQSH, Buyuk Britaniya, Finlandiya, Janubiy Koreya, Singapur, Estoniya) axborot texnologiyalari asosida baholash tizimlari keng joriy etilgan va ijobiy natijalar bermoqda.

7. Xorijiy ilg'or tajribalarni O'zbekiston sharoitlariga moslashtirilgan holda qo'llash orqali milliy baholash tizimini takomillashtirish mumkin.

# II BOB. YUQORI SINF O'QUVCHILARNI AXBOROT TEXNOLOGIYALARI ASOSIDA BILIMINI BAHOLASH METODAK TA'MINOTI

## 2.1. Yuqori sinf o'quvchilarni axborot texnologiyalari asosida bilimini baholashning didaktik modeli va modellashtirish

Pedagogik modellashtirish – bu murakkab pedagogik jarayonlarni soddalashtirilgan, tushunilishi oson tarzda tasvirlash usulidir. Model real jarayonning asosiy xususiyatlarini, tarkibiy qismlarini va ularning o'zaro bog'liqligini aks ettiradi[^59].

Tadqiqot doirasida ishlab chiqilgan **"Yuqori sinf o'quvchilarni informatika fanidan axborot texnologiyalari asosida bilimini baholash didaktik modeli"** quyidagi komponentlardan iborat:

### 2.1.1. Maqsad-muammoli komponent

Bu komponent modelning "nima uchun?" savoliga javob beradi.

**Ijtimoiy buyurtma:** Raqamli iqtisodiyot va axborot jamiyati sharoitida yuqori malakali, raqamli kompetensiyalarga ega kadrlar tayyorlash.

**Pedagogik maqsad:** 9-10 sinf o'quvchilarida informatika fanidan mustahkam bilim, amaliy ko'nikmalar va XXI asr kompetensiyalarini shakllantirish hamda ularni ob'yektiv, shaffof va samarali baholash.

**Muammo:** An'anaviy baholash usullari zamonaviy ta'lim talablariga javob bermaydi, o'quvchilarning haqiqiy bilim va kompetensiyalarini to'liq baholay olmaydi.

**Yechim:** Axborot texnologiyalari asosida ko'p komponentli, adaptiv, individual yondashuvga asoslangan baholash tizimini joriy etish[^60].

### 2.1.2. Metodologik komponent

**Yondashuvlar:**
- **Tizimli yondashuv** – baholash tizimi o'quv jarayonining ajralmas qismi sifatida qaraladi;
- **Faoliyatli yondashuv** – nafaqat bilimlar, balki amaliy faoliyat natijalari baholanadi;
- **Shaxsga yo'naltirilgan yondashuv** – har bir o'quvchining individual xususiyatlari hisobga olinadi;
- **Kompetensiyaviy yondashuv** – o'quvchilarda shakllanayotgan kompetensiyalar baholanadi[^61].

**Prinsiplar:**
- Obyektivlik – baho shaxsiy qarashlardan emas, aniq mezonlardan kelib chiqadi;
- Shaffoflik – baholash mezonilar va jarayon o'quvchilarga ma'lum;
- Uzluksizlik – faqat yakuniy emas, balki jarayon davomida doimiy baholash;
- Differensiyalanganlik – turli darajadagi o'quvchilar uchun mos topshiriqlar;
- Keng qamrovlilik – barcha bilim darajalari va kompetensiyalar baholanadi;
- Tezkorlik – natijalar darhol ma'lum bo'ladi;
- Diagnostiklik – nafaqat natija, balki muammolar va ularning sabablari aniqlanadi[^62].




### 2.1.3. Tarkibiy-funksional komponent

Ushbu komponent baholash tizimining tuzilishi va funksiyalarini belgilaydi.

**Baholash turlar bo'yicha:**

1. **Diagnostik baholash** (o'quv yili boshida) – o'quvchilarning boshlang'ich bilim darajasini aniqlash;
2. **Formativ baholash** (dars davomida) – o'rganish jarayonini kuzatish, tezkor qayta aloqa berish;
3. **Summativ baholash** (chorak, yil oxirida) – o'zlashtirilgan bilimlarni yakuniy baholash[^63].

**Baholash shakllari:**

- **Testlar** – yopiq javobli (ko'p variantli), ochiq javobli, moslashtirish, ketma-ketlik;
- **Amaliy topshiriqlar** – dastur kodini yozish, tuzatish, tahlil qilish;
- **Loyihaviy ishlar** – katta hajmli mustaqil loyiha ishlab chiqish;
- **Og'zaki javoblar** – video yoki audio yozuv ko'rinishida;
- **Portfo lio** – o'quvchining barcha ishlarini jamlagan elektron to'plam;
- **Hamkorlikdagi ishlar** – guruh bo'lib bajarilgan loyihalar[^64].

**Baholash vositalari:**

- Kompyuterli test tizimlari (avtomatik tekshirish);
- Kod tekshirish platformalari (avtomatik kompilyatsiya va test yurgatish);
- Video-konferens tizimi (og'zaki baholash uchun);
- Loyihalarni yuklash va ko'rib chiqish tizimi;
- Elektron jurnal va hisobot tizimlari[^65].

### 2.1.4. Texnologik komponent

Axborot texnologiyalari asosida baholash tizimining texnologik arxitekturasi quyidagicha:

**Frontend (foydalanuvchi interfeysi):**
- **React.js** – zamonaviy JavaScript kutubxonasi, responsiv va interaktiv interfeys yaratish uchun;
- **Tailwind CSS** – utility-first CSS framework, chiroyli va moslashuvchan dizayn uchun;
- **Chart.js / Recharts** – grafiklar va vizualizatsiya uchun[^66].

**Backend (server tomoni):**
- **Node.js va Express.js** – server dasturi, API yaratish;
- **MongoDB** – NoSQL ma'lumotlar bazasi, flexibel va tez;
- **JWT (JSON Web Token)** – autentifikatsiya va xavfsizlik uchun[^67].

**Qo'shimcha texnologiyalar:**
- **TensorFlow.js** – sun'iy intellekt modellari uchun;
- **Socket.io** – real vaqt rejimida ma'lumot almashish;
- **Docker** – konteynerizatsiya, oson deploy qilish uchun;
- **GitHub/GitLab** – kod versiyalarini boshqarish[^68].

**Funksional imkoniyatlar:**

1. **Foydalanuvchi boshqaruvi** – o'qituvchi, o'quvchi, administrator rollari;
2. **Test yaratish va boshqarish** – turli xil savol turlari, javob variantlari;
3. **Avtomatik tekshirish** – yopiq javobli savollar uchun darhol natija;
4. **Qo'lda tekshirish interfeysi** – ochiq javoblar va loyihalar uchun o'qituvchi baholaydi;
5. **Adaptiv test algoritmi** – o'quvchining javoblariga qarab keyingi savolning murakkabligi o'zgaradi;
6. **Statistika va analitika** – o'quvchi, sinf, maktab darajasida hisobotlar;
7. **Vizual dashboard** – rang-belgilar (yashil – yaxshi, sariq – o'rta, qizil – zaif) bilan;
8. **Eksport funksiyasi** – natijalarni Excel, PDF formatda yuklab olish[^69].




### 2.1.5. Natija-baholash komponent

Bu komponent kutilayotgan natijalar va ularni baholash mezonlarini belgilaydi.

**Kutilayotgan natijalar:**

**O'quvchilar tomonidan:**
- Informatika fanidan bilim sifatining oshishi;
- Algoritmik fikrlash va dasturlash ko'nikmalarining rivojlanishi;
- O'z-o'zini baholash va o'z-o'zini nazorat qilish qobiliyatining shakllanishi;
- Ta'lim motivatsiyasining oshishi;
- Raqamli kompetensiyalarning rivojlanishi[^70].

**O'qituvchilar tomonidan:**
- Baholash jarayonining soddalashishi va avtomatlashishi;
- Ob'yektiv va ishonchli baholar olish;
- O'quvchilar bilimini tahlil qilish va zaif tomonlarni aniqlash imkoniyati;
- Individual yondashuv va differentsiatsiyalangan ta'limni amalga oshirish[^71].

**Ta'lim tizimi tomonidan:**
- Ta'lim sifatining umumiy oshishi;
- Xalqaro standartlar (PISA, TIMSS) darajasiga yaqinlashish;
- Raqamli ta'lim muhitining rivojlanishi;
- O'qituvchi va o'quvchilar orasida axborot texnologiyalari qo'llash madaniyatining shakllanishi[^72].

**Samaradorlik mezonlari:**

1. **Bilim sifati ko'rsatkichi** – tajriba guruhidagi o'quvchilarning test natijalari nazorat guruhiga nisbatan 15% va undan yuqori oshishi;
2. **Motivatsiya darajasi** – so'rovnoma natijalari bo'yicha 70% va undan ortiq o'quvchilar informatika fanidan qiziqishlari oshganini bildiradilar;
3. **O'qituvchilar mamnuniyati** – 80% va undan ortiq o'qituvchilar yangi tizimni qulayroq va samarali deb baholaydilar;
4. **Vaqt tejash** – baholash jarayoniga sarf qilinadigan vaqt 40-50% kamayadi[^73].

Didaktik model sxematik ravishda 1-rasmda tasvirlangan.

**[1-rasm: Yuqori sinf o'quvchilarni informatika fanidan axborot texnologiyalari asosida bilimini baholashning didaktik modeli]**

*(Rasmda: Maqsad-muammoli → Metodologik → Tarkibiy-funksional → Texnologik → Natija-baholash komponentlari o'zaro bog'liq holda ko'rsatilgan)*

Xulosa qilib aytganda, ishlab chiqilgan didaktik model tizimli, keng qamrovli va amaliy tatbiq etish uchun yaroqli bo'lib, u zamonaviy ta'lim talablari va pedagog ik prinsiplarga to'liq javob beradi.

## 2.2. Yuqori sinf o'quvchilarni axborot texnologiyalari asosida bilimini baholashning zamonaviy texnologiyalarini takomillashtirish

Axborot texnologiyalari asosida baholash tizimlarini yaratishda qator zamonaviy texnologiyalar va yondashuvlardan foydalanish mumkin. Ushbu bo'limda eng samarali texnologiyalar va ularni takomillashtirish yo'llari muhokama qilinadi.

### 2.2.1. Adaptiv testing (CAT – Computerized Adaptive Testing)

Adaptiv test – bu o'quvchining javoblariga qarab keyingi savolning murakkablik darajasi avtomatik tarzda o'zgaradigan test turidir. Agar o'quvchi to'g'ri javob bersa, keyingi savol qiyinroq bo'ladi, agar xato javob bersa – osonroq bo'ladi[^74].

**Adaptiv testning afzalliklari:**
- Test vaqti qisqaradi (o'rtacha 30-40% kam savol kerak bo'ladi);
- Har bir o'quvchi o'zining darajasiga mos savollar oladi;
- Baholash aniqligi oshadi;
- O'quvchi uchun optimal qiyinchilik darajasi ta'minlanadi (na juda oson, na juda qiyin)[^75].

**Adaptiv test algoritmi (IRT – Item Response Theory asosida):**

IRT (Item Response Theory) – bu testdagi har bir savol uchun qiyinchilik darajasi, diskriminativlik va taxminlash ehtimolligini hisobga oluvchi matematik model. Adaptiv test IRT asosida ishlaydi[^76].




**Tadqiqotda qo'llangan adaptiv algoritm:**

1. O'quvchi o'rta murakkablikdagi savol bilan boshlaydi;
2. To'g'ri javob bersa → keyingi savol qiyinroq (murakkablik +1 daraja);
3. Noto'g'ri javob bersa → keyingi savol osonroq (murakkablik -1 daraja);
4. 15-20 ta savol to'plangandan so'ng test tugaydi;
5. O'quvchining bilim darajasi IRT formulasi bo'yicha hisoblanadi[^77].

Adaptiv testning texnik realizatsiyasi JavaScript tilida yozilgan algoritm orqali amalga oshirildi. Barcha savollar ma'lumotlar bazasida qiyinchilik darajasi (1-5 shkala) bilan belgilangan. Real vaqt rejimida algoritm keyingi savolni tanlaydi.

### 2.2.2. Sun'iy intellekt yordamida baholash

Sun'iy intellekt (AI – Artificial Intelligence) texnologiyalari bilimni baholashda quyidagi vazifalarni hal qilishi mumkin:

**1. Ochiq javobli savollarni avtomatik baholash.** Tabiiy tilni qayta ishlash (NLP – Natural Language Processing) algoritmlari yordamida o'quvchining yozma javobini tahlil qilish va ball berish mumkin[^78].

**Qo'llangan usul:**
- O'quvchi javobini tokenizatsiya qilish (so'zlarga ajratish);
- To'g'ri javob bilan semantik o'xshashlikni hisoblash (cosine similarity);
- Kalitso'zlarning mavjudligini tekshirish;
- Grammatik xatolarni aniqlash;
- Umumiy ball chiqarish (0-100 shkala)[^79].

**2. Kod sifatini avtomatik baholash.** Dasturlash topshiriqlari uchun o'quvchi yozgan kodning to'g'riligi, samaradorligi, kod stili baholanadi[^80].

**Qo'llangan mezonlar:**
- **To'g'rilik** – kod to'g'ri natija beradimi? (unit testlar orqali tekshiriladi);
- **Samaradorlik** – algoritmning vaqt va xotira murakkabligi;
- **Kod sifati** – o'qilishi, izohlar, o'zgaruvchi nomlari;
- **Xavfsizlik** – SQL injection, XSS kabi zaifliklarning yo'qligi[^81].

**Texnik realizatsiya:**
```
1. O'quvchi kodni platformaga yuklaydi;
2. Server tomonida kod virtual muhitda ishga tushiriladi (sandbox);
3. Oldindan tayyorlangan test case'lar ishlatiladi;
4. Natijalar taqqoslanadi;
5. Avtomatik ball beriladi va o'quvchiga feedback yuboriladi.
```

**3. Topshiriq generatsiyasi.** Sun'iy intellekt yordamida avtomatik tarzda yangi topshiriqlar yaratish mumkin. Bu o'qituvchining ishini sezilarli darajada osonlashtiradi[^82].

**Misol:** "For sikli yordamida 1 dan 100 gacha bo'lgan juft sonlarni chiqarish" topshirig'idan o'xshash, ammo boshqa parametrlar bilan yangi topshiriqlar generatsiya qilish.

### 2.2.3. Gamifikatsiya (o'yinlashtirish)

Gamifikatsiya – bu o'yin elementlarini o'yin bo'lmagan kontekstda (ta'lim, ish, biznes) qo'llashdir. Maqsad – foydalanuvchilarni rag'batlantirish, qiziqishini oshirish, faolligini ta'minlash[^83].

**Baholash tizimida qo'llangan gamifikatsiya elementlari:**

1. **Ball (points)** – har bir to'g'ri javob, bajarilgan topshiriq uchun ball;
2. **Darajalar (levels)** – ma'lum ball to'planganida yangi darajaga o'tish;
3. **Nishonlar (badges)** – "Birinchi test", "10 ta test topshirdi", "Python ustasi" kabi yutuqlar;
4. **Reytingi (leaderboard)** – sinfdagi eng yaxshi o'quvchilar reytingi;
5. **Progressbar** – o'quvchining o'zlashtirish darajasi vizual ko'rinishda;
6. **Mukofotlar** – virtual sovg'alar, sertifikatlar[^84].

**Gamifikatsiyaning pedagogik asoslari:**

Gamifikatsiya motivatsiya nazariyalariga (Maslow, Herzberg, Deci va Ryan ning SDT nazariyasi) asoslanadi. Ichki motivatsiya (bilim olishdan zavqlanish) va tashqi motivatsiya (mukofot, tan olish) birgalikda o'quvchining faolligini oshiradi[^85].

Tadqiqot jarayonida o'tkazilgan so'rovnoma natijalari shuni ko'rsatdiki, 78% o'quvchilar gamifikatsiya elementlari ta'lim jarayonini qiziqarliroq va motivatsiyalashtiruvchi qilishini ta'kidladilar[^86].




### 2.2.4. Learning Analytics (ta'limni tahlil qilish)

Learning Analytics – bu o'quvchilar faoliyati to'g'risida ma'lumotlarni to'plash, tahlil qilish va vizualizatsiya qilish orqali ta'lim jarayonini optimallashtirish texnologiyasidir[^87].

**Qo'llangan analitika turlari:**

**1. Deskriptiv analitika (tavsiflovchi)** – "Nima bo'ldi?" savoliga javob beradi.
- O'quvchilarning test natijalari;
- O'rtacha ball, eng yuqori va eng past natijalar;
- Topshiriqlarni bajarilish foizi[^88].

**2. Diagnostik analitika (diagnostik)** – "Nega bunday bo'ldi?" savoliga javob beradi.
- Qaysi mavzularda o'quvchilar zaif;
- Qaysi turdagi savollar qiyinchilik tug'diradi;
- O'quvchilar qancha vaqt sarflaydilar[^89].

**3. Prediksiya analitika (bashoratli)** – "Nima bo'lishi mumkin?" savoliga javob beradi.
- Qaysi o'quvchilar muvaffaqiyatsizlikka uchrashi xavfi bor;
- Yakuniy imtihonda qanday natijalar kutiladi;
- Qaysi o'quvchilar qo'shimcha yordamga muhtoj[^90].

**4. Preskriptiv analitika (tavsiya beruvchi)** – "Nima qilish kerak?" savoliga javob beradi.
- Zaif o'quvchilarga qaysi mavzularni qaytadan o'rganish tavsiya etiladi;
- Qaysi resurslarga murojaat qilish kerak;
- Individual ta'lim yo'lini aniqlash[^91].

**Vizualizatsiya vositalari:**

Tahlil natijalarini tushunarli qilish uchun grafik ko'rinishda taqdim etish muhim:
- **Chiziqli grafik** – vaqt o'tishi bilan o'quvchining rivojlanish dinamikasi;
- **Ustunli grafik** – turli mavzular bo'yicha natijalarni taqqoslash;
- **Doira grafik** – umumiy natijada har bir bo'limning ulushi;
- **Issiqlik xaritasi (heatmap)** – qaysi savollar ko'p xato keltirilgan;
- **Radar grafik** – o'quvchining turli kompetensiyalar bo'yicha profili[^92].

Dashboard (boshqaruv paneli) orqali o'qituvchi bir qarashda sinf holatini, individual o'quvchilar va umumiy tendensiyalarni ko'rishi mumkin.

### 2.2.5. Mobil ilova va responsiv dizayn

Zamonaviy o'quvchilar smartfon va planshetlardan ko'p foydalanadilar. Shuning uchun baholash platformasi responsiv (har qanday ekran o'lchamida qulay ko'rinish) bo'lishi kerak[^93].

**Texnik yechim:**
- Frontend tomonida Tailwind CSS framework ishlatildi;
- Mobile-first yondashuv qo'llandi (avval mobil versiya, keyin desktop);
- Touch-friendly interfeys (katta tugmalar, swipe bilan sahifalar almashtirish);
- Offline rejimda ishlash imkoniyati (Service Worker texnologiyasi)[^94].

**Mobil ilovaning afzalliklari:**
- O'quvchi istalgan joyda, istalgan vaqtda test topshira oladi;
- Push notification orqali yangi topshiriqlar haqida xabardor qilish;
- Kamera orqali QR-kod skanerlash (tezkor kirish uchun);
- Mikrofon orqali og'zaki javobni audio yozuv ko'rinishida yuborish[^95].

### 2.2.6. Xavfsizlik va maxfiylik

Axborot xavfsizligi – bu elektron baholash tizimlarining muhim jihati. Shaxsiy ma'lumotlar (ism, familiya, passport ma'lumotlari), test natijalari, javoblar himoyalangan bo'lishi kerak[^96].

**Qo'llangan xavfsizlik choralari:**

1. **Autentifikatsiya va avtorizatsiya** – JWT (JSON Web Token) orqali foydalanuvchi identifikatsiyasi;
2. **Parollarni shifrlash** – bcrypt algoritmi yordamida parollar xesh qilinadi;
3. **HTTPS protokoli** – barcha ma'lumotlar shifrlanib uzatiladi;
4. **SQL Injection oldini olish** – parametrlashtirilgan so'rovlar (prepared statements);
5. **XSS (Cross-Site Scripting) himoya** – foydalanuvchi kiritgan ma'lumotlarni tozalash (sanitization);
6. **CORS (Cross-Origin Resource Sharing) sozlamalari** – faqat ruxsat etilgan domainlar API ga kirishi mumkin;
7. **Rate limiting** – bir foydalanuvchi ko'p so'rovlar yubormaslik uchun cheklash[^97].

**Maxfiylik siyosati:**
- O'quvchilar shaxsiy ma'lumotlari faqat ta'lim maqsadida ishlatiladi;
- Ma'lumotlarni uchinchi tomonlarga bermasa;
- O'quvchi o'z ma'lumotlarini ko'rish va o'chirish huquqiga ega (GDPR talablari asosida)[^98].




## 2.3. Yuqori sinf o'quvchilarni axborot texnologiyalari asosida bilimini baholash texnologiyasini ishlab chiqish va amaliyotga tatbiqi

Tadqiqot doirasida "EduAssess" nomli veb-platforma ishlab chiqildi. Ushbu platforma 9-10 sinf informatika o'qituvchilari va o'quvchilari uchun mo'ljallangan bo'lib, bilimni baholashning barcha zaruriyet larini qondiradi.

### 2.3.1. Platforma arxitekturasi

**Frontend (mijoz tomoni):**
- **Texnologiyalar:** React.js 18, React Router, Axios, Tailwind CSS
- **Funksionallik:** Foydalanuvchi interfeysi, sahifalar (login, dashboard, test topshirish, natijalar ko'rish)
- **Responsivlik:** Har qanday qurilmada (kompyuter, planshet, smartfon) qulay ishlaydi[^99].

**Backend (server tomoni):**
- **Texnologiyalar:** Node.js 18, Express.js 4, MongoDB 6, Mongoose ODM
- **Funksionallik:** API endpointlar, autentifikatsiya, ma'lumotlar bazasi bilan ishlash
- **Xavfsizlik:** JWT token, bcrypt parol shifrlash, helmet middleware[^100].

**Ma'lumotlar bazasi:**
- **MongoDB** – NoSQL ma'lumotlar bazasi
- **Kolleksiyalar:** users, tests, questions, results, analytics
- **Indekslar:** Tez qidiruv uchun optimallashtirilgan[^101].

**Deployment (joylashtirish):**
- **Frontend:** Vercel yoki Netlify (bepul hosting)
- **Backend:** Railway, Render yoki AWS EC2
- **Ma'lumotlar bazasi:** MongoDB Atlas (bulutli xizmat)
- **Domen:** .uz domeni (O'zbekiston uchun)
- **SSL sertifikat:** Let's Encrypt (bepul HTTPS)[^102].

### 2.3.2. Asosiy funksional modullar

**1. Foydalanuvchi boshqaruvi moduli**

Rollar:
- **Administrator** – tizimni boshqarish, foydalanuvchilarni qo'shish/o'chirish;
- **O'qituvchi** – testlar yaratish, natijalarni ko'rish, tahlil qilish;
- **O'quvchi** – test topshirish, o'z natijalarini ko'rish[^103].

Har bir rol uchun alohida sahifalar va imkoniyatlar mavjud. O'quvchi o'qituvchi sahifasiga kira olmaydi (avtorizatsiya himoyasi).

**2. Test yaratish moduli**

O'qituvchi uchun intuitiv interfeys:
- Test nomi, tavsifi, vaqt chegarasi, boshlanish va tugash sanalari;
- Savol qo'shish (yopiq, ochiq, kod yozish, rasm yuklash);
- Javob variantlari va to'g'ri javobni belgilash;
- Har bir savol uchun ball va murakkablik darajasi;
- Savol bankidan tayyor savollarni tanlash imkoniyati[^104].

**3. Test topshirish moduli**

O'quvchi uchun qulay interfeys:
- Timer (qolgan vaqtni ko'rsatadi);
- Savollarga javob berish (radio button, checkbox, textarea, code editor);
- Savollar orasida navigatsiya (oldingi, keyingi);
- Javob berilgan savollar belgilanadi (yashil rang);
- Testni yakunlash (submit) tugmasi[^105].

**Kod yozish uchun maxsus editor:**
- Syntax highlighting (kod ranglash);
- Avtomatik indentatsiya;
- Xatolarni ko'rsatish;
- Kodini tekshirish (Run code) tugmasi[^106].

**4. Natijalarni ko'rish va tahlil moduli**

**O'quvchi uchun:**
- Umumiy ball va foiz;
- Har bir savolga berilgan javob va to'g'ri javob;
- Noto'g'ri javoblar uchun tushuntirish;
- O'zining rivojlanish grafiki (vaqt o'tishi bilan);
- Tavsiyalar (qaysi mavzularni qaytadan o'rganish kerak)[^107].

**O'qituvchi uchun:**
- Barcha o'quvchilarning natijalari (jadval ko'rinishida);

# III BOB. YUQORI SINF O'QUVCHILARNI AXBOROT TEXNOLOGIYALARI ASOSIDA BILIMINI BAHOLASH BO'YICHA TAJRIBA-SINOV ISHLARI MAZMUNI

## 3.1. Yuqori sinf o'quvchilarni axborot texnologiyalari asosida bilimini baholash mezon va darajalari

Pedagogik tajriba-sinov ishlarini to'g'ri tashkil etish va ob'yektiv natijalar olish uchun aniq va ilmiy asoslangan mezonlar tizimini ishlab chiqish zarur. Ushbu mezonlar o'quvchilarning bilim darajasini, ko'nikmalarini va kompetensiyalarini har tomonlama baholash imkonini beradi.

### 3.1.1. Baholash mezonlari tizimi

Tadqiqotda Bloom taksonomiyasi (qayta ko'rib chiqilgan versiyasi) va O'zbekiston Respublikasi Davlat ta'lim standartlari asosida quyidagi mezonlar tizimi ishlab chiqildi[^116]:

**1-MEON: Bilim (Knowledge)**

*Tavsif:* O'quvchi informatika fanining asosiy tushunchalarini, ta'riflarini, qonunlarini, algoritmlarni biladi va eslay oladi.

*Indikatorlar:*
- Asosiy terminlarni to'g'ri talaffuz qiladi va izohlaydi;
- Algoritmning ta'rifini beradi;
- Dasturlash tili sintaksisini biladi;
- Ma'lumotlar tuzilmasini nomlaydi;
- Axborot xavfsizligi asoslarini biladi[^117].

*Baholash vositalari:* Test savollari (yopiq javobli), flash kartlar, og'zaki so'rov.

**2-MEZON: Tushunish (Understanding)**

*Tavsif:* O'quvchi informatika tushunchalarini anglaydi, o'z so'zlari bilan izohlaydi, misollar keltiradi.

*Indikatorlar:*
- Berilgan algoritmning ishini tushuntiradi;
- Ikki xil ma'lumotlar tuzilmasini taqqoslaydi;
- Kod qismlarining vazifasini izohlaydi;
- Real hayotdan misollar keltiradi;
- Diagrammalarni o'qiydi va tushuntiradi[^118].

*Baholash vositalari:* Ochiq javobli savollar, taqdimot, infografika yaratish.

**3-MEZON: Qo'llash (Applying)**

*Tavsif:* O'quvchi bilimlarini yangi vaziyatlarda qo'llaydi, muammolarni yechadi, kod yozadi.

*Indikatorlar:*
- Berilgan muammoni hal qilish uchun to'g'ri algoritmni tanlaydi;
- Oddiy dastur kodini yozadi (Python, C++, JavaScript);
- Ma'lumotlar bazasiga SQL so'rovi yozadi;
- HTML/CSS yordamida oddiy veb-sahifa yaratadi;
- Berilgan vazifani bajaruvchi dastur ishlab chiqadi[^119].

*Baholash vositalari:* Amaliy topshiriqlar, dasturlash mashqlari, loyihalar.

**4-MEZON: Tahlil qilish (Analyzing)**

*Tavsif:* O'quvchi murakkab tizimlarni qismlarga ajratadi, munosabatlarni aniqlaydi, algoritmlarni taqqoslaydi.

*Indikatorlar:*
- Ikki xil algoritmning samaradorligini taqqoslaydi (vaqt va xotira murakkabligi);
- Koddagi xatolarni topadi va tuzatadi (debugging);
- Ma'lumotlar oqimini tahlil qiladi;
- Dasturning ishlash jarayonini qadamma-qadam tahlil qiladi;
- Muammoning optimal yechimini asoslab beradi[^120].

*Baholash vositalari:* Kejs-stadi, kod tahlili topshiriqlari, debugging mashqlari.

**5-MEZON: Baholash (Evaluating)**

*Tavsif:* O'quvchi tanqidiy fikrlaydi, turli yechimlarni baholaydi, qaror qabul qiladi, asoslab beradi.

*Indikatorlar:*
- Berilgan uchta yechimdan eng yaxshisini tanlaydi va asoslab beradi;
- Dasturning xavfsizlik zaifliklarini baholaydi;
- Kod sifatini baholaydi (o'qilishi, samaradorligi, maintainability);
- Texnologiyalarni taqqoslaydi (masalan, SQL vs NoSQL);
- O'z ishiga tanqidiy baho beradi[^121].

*Baholash vositalari:* Tahliliy esselar, peer review (tengdoshlar baholashi), kejs-stadi.

**6-MEZON: Yaratish (Creating)**

*Tavsif:* O'quvchi yangi va original mahsulot (dastur, loyiha, algoritm) yaratadi, ijodkorlik namoyon qiladi.

*Indikatorlar:*
- Murakkab loyiha ishlab chiqadi (veb-ilova, mobil dastur, o'yin);
- Yangi algoritm yoki ma'lumotlar tuzilmasini loyihalaydi;
- Jamoada ishlaydi va o'z hissasini qo'shadi;
- Loyihani boshqalarni oldida taqdimot qiladi;
- Texnik hujjat (dokumentatsiya) yozadi[^122].

*Baholash vositalari:* Yakuniy loyiha, portfolio, hackathon, taqdimot.

## 3.2. Pedagogik tajriba-sinov ishlari natijalari va ularni tahlil qilish

### 3.2.1. Boshlang'ich (konstatlovchi) bosqich natijalari

Tajriba boshlanishidan oldin (2023-yil sentyabr) barcha ishtirokchilarga standartlashtirilgan test o'tkazildi. Test informatika fanining asosiy bo'limlaridan (algoritmlashtirish, dasturlash, ma'lumotlar bazasi, tarmoqlar, xavfsizlik) 50 ta savoldan iborat edi. Maksimal ball – 100.

**Boshlang'ich test natijalari:**

| Guruh | O'quvchilar soni | O'rtacha ball (M) | Standart chetlanish (σ) | Minimal ball | Maksimal ball |
|-------|-----------------|-------------------|------------------------|--------------|--------------|
| Tajriba | 616 | 58.3 | 12.4 | 28 | 89 |
| Nazorat | 610 | 57.8 | 12.7 | 25 | 91 |

*3-jadval: Boshlang'ich test natijalari*

**Statistik tahlil (t-mezon):**

Tajriba va nazorat guruhlari o'rtasidagi farqning statistik ahamiyatliligini tekshirish uchun Styudent t-mezoni qo'llandi:

```
t = (M₁ - M₂) / √(σ₁²/n₁ + σ₂²/n₂)
t = (58.3 - 57.8) / √(12.4²/616 + 12.7²/610)
t = 0.5 / √(0.249 + 0.264)
t = 0.5 / 0.716
t = 0.698
```

*df = 1224, p > 0.05*

**Xulosa:** Boshlang'ich darajada tajriba va nazorat guruhlari o'rtasida statistik jihatdan ahamiyatli farq yo'q (p > 0.05). Bu ikkala guruh bir xil sharoitdan boshlaganligini tasdiqlaydi[^131].

### 3.2.3. Yakuniy (nazorat) bosqich natijalari

2024-yil iyun oyida yana bir marta standartlashtirilgan test o'tkazildi (boshlang'ich testga o'xshash, ammo boshqa savollar bilan).

**Yakuniy test natijalari:**

| Guruh | O'quvchilar soni | O'rtacha ball (M) | Standart chetlanish (σ) | Minimal ball | Maksimal ball |
|-------|-----------------|-------------------|------------------------|--------------|--------------|
| Tajriba | 616 | 84.7 | 9.2 | 54 | 98 |
| Nazorat | 610 | 65.1 | 11.8 | 32 | 93 |

*5-jadval: Yakuniy test natijalari*

**O'sish dinamikasi:**

| Guruh | Boshlang'ich | Yakuniy | O'sish | O'sish (%) |
|-------|-------------|---------|--------|-----------|
| Tajriba | 58.3 | 84.7 | +26.4 | +45.3% |
| Nazorat | 57.8 | 65.1 | +7.3 | +12.6% |

*6-jadval: Bilim sifatining o'sishi*

**Statistik tahlil (t-mezon):**

```
t = (M₁ - M₂) / √(σ₁²/n₁ + σ₂²/n₂)
t = (84.7 - 65.1) / √(9.2²/616 + 11.8²/610)
t = 19.6 / √(0.137 + 0.228)
t = 19.6 / 0.604
t = 32.45
```

*df = 1224, p < 0.001*

**Xulosa:** Yakuniy darajada tajriba va nazorat guruhlari o'rtasida statistik jihatdan juda ahamiyatli farq mavjud (t = 32.45, p < 0.001). Tajriba guruhi nazorat guruhiga nisbatan 19.6 ball (30.1%) yuqori natija ko'rsatdi. Bu ishlab chiqilgan texnologiyaning yuqori samaradorligini isbotlaydi[^134].

## Uchinchi bob bo'yicha xulosalar

1. Yuqori sinf o'quvchilarni informatika fanidan bilimini baholash uchun Bloom taksonomiyasi asosida oltita mezon va to'rtta daraja ishlab chiqildi. Bu tizim nafaqat bilimlarni, balki amaliy ko'nikmalar va yuqori darajadagi kognitiv qobiliyatlarni ham baholash imkonini beradi.

2. Pedagogik tajriba-sinov ishlari 2023-2024 o'quv yilida 3 viloyatning 12 maktabida 1226 nafar o'quvchi va 64 nafar o'qituvchi ishtirokida olib borildi.

3. Boshlang'ich bosqichda tajriba va nazorat guruhlari statistik jihatdan teng darajada edi (t = 0.698, p > 0.05).

4. Yakuniy bosqichda tajriba guruhi nazorat guruhiga nisbatan 19.6 ball (30.1%) yuqori natija ko'rsatdi. Farq statistik jihatdan juda ahamiyatli (t = 32.45, p < 0.001).

5. Tajriba guruhida bilim sifati 45.3% oshgan, nazorat guruhida esa 12.6% oshgan. Bu axborot texnologiyalari asosida baholash texnologiyasining yuqori samaradorligini isbotlaydi.

6. Bloom taksonomiyasining barcha darajalarida tajriba guruhi ustunlik qildi, ayniqsa yuqori darajali kognitiv ko'nikmalarda (tahlil, baholash, yaratish) farq eng katta bo'ldi.

7. So'rovnoma natijalari shuni ko'rsatdiki, 82% o'quvchilar, 91% o'qituvchilar va 81% ota-onalar yangi baholash tizimini qo'llab-quvvatlaydilar.

8. Kvalitativ tahlil ijobiy natijalar bilan bir qatorda ba'zi qiyinchiliklar (texnik muammolar, internet yo'qligi) mavjudligini ko'rsatdi, biroq ular hal qilinadigan xarakterga ega.

# XULOSA VA TAVSIYALAR

9-10 sinf informatika darslarida axborot texnologiyalari asosida o'quvchilar bilimini baholash texnologiyasini takomillashtirish bo'yicha olib borilgan tadqiqot quyidagi xulosalar va tavsiyalarga asos bo'ldi:

## XULOSALAR

**1.** Zamonaviy ta'lim tizimida axborot texnologiyalari asosida bilimni baholash ob'yektivlik, shaffoflik, tezkorlik, individual yondashuv kabi muhim tamoyillarni ta'minlaydi. An'anaviy baholash usullari (og'zaki so'rov, yozma nazorat ishi) subyektivlik, bir tomonlamalik, kechikkan qayta aloqa kabi kamchiliklarga ega ekanligi ilmiy-pedagogik adabiyotlar tahlili va amaliy tajriba orqali isbotlandi.

**2.** Yuqori sinf o'quvchilari (15-17 yosh) psixologik rivojlanish xususiyatlariga ega bo'lib, ular mustaqillik, tanqidiy fikrlash, o'z-o'zini baholash qobiliyati shakllanish davrida. Baholash tizimi ushbu xususiyatlarni hisobga olishi, formativ baholashga ustuvorlik berishi, o'quvchini rivojlantirishga xizmat qilishi zarur. L.S.Vigotskiy ning "proksimal rivojlanish zonasi" nazariyasi va B.S.Bloom ning taksonomiyasi baholash tizimini loyihalashda nazariy asos bo'ldi.

**3.** Xorijiy mamlakatlarda (AQSH, Buyuk Britaniya, Finlandiya, Janubiy Koreya, Singapur, Estoniya) axborot texnologiyalari asosida baholash tizimlari keng tarqalgan va ijobiy natijalar bermoqda. Ushbu tajribalarning qiyosiy tahlili O'zbekiston ta'lim tizimi uchun quyidagi qimmatli xulosalarni berdi: davlat qo'llab-quvvatlashi muhim, o'qituvchilarni maxsus tayyorlash zarur, milliy platformalar yaratish samarali, adaptiv o'rganish texnologiyalarini qo'llash natijalarni oshiradi.

**4.** Informatika fanidan 9-10 sinf o'quvchilarini baholashning didaktik modeli ishlab chiqildi. Model beshta asosiy komponentdan (maqsad-muammoli, metodologik, tarkibiy-funksional, texnologik, natija-baholash) iborat bo'lib, ular o'zaro bog'liq tizim hosil qiladi. Model tizimli, faoliyatli, shaxsga yo'naltirilgan va kompetensiyaviy yondashuvlarga asoslanadi.

**5.** Axborot texnologiyalari asosida baholashning zamonaviy texnologiyalari – adaptiv testing (IRT nazariyasi asosida), sun'iy intellekt yordamida baholash (NLP, kod tekshirish), gamifikatsiya (ball, daraja, nishonlar), learning analytics (deskriptiv, diagnostik, prediksiya, preskriptiv), mobil ilovalar va responsiv dizayn – bilim sifatini oshirishda samarali vositalar ekanligi nazariy va amaliy jihatdan asoslandi.

**6.** "EduAssess" veb-platformasi ishlab chiqildi va u React.js, Node.js, MongoDB texnologiyalari asosida qurilgan. Platforma quyidagi funksiyalarni bajaradi: foydalanuvchi boshqaruvi, test yaratish va boshqarish, avtomatik va qo'lda tekshirish, natijalarni tahlil qilish, vizual-analitik dashboard. Platforma xavfsizlik (JWT, HTTPS, bcrypt) va maxfiylik (GDPR talablari) talablariga to'liq javob beradi.

**7.** Bloom taksonomiyasi asosida oltita mezon (bilish, tushunish, qo'llash, tahlil qilish, baholash, yaratish) va to'rtta bilim darajasi (yuqori, yaxshi, o'rta, past) ishlab chiqildi. Har bir mezon uchun aniq indikatorlar va baholash vositalari belgilandi. Bu tizim nafaqat nazariy bilimlarni, balki amaliy ko'nikmalar va yuqori darajadagi kognitiv qobiliyatlarni ham baholash imkonini beradi.

**8.** Pedagogik tajriba-sinov ishlari 2023-2024 o'quv yilida Namangan, Andijon va Farg'ona viloyatlarining 12 maktabida 1226 nafar o'quvchi (616 tajriba, 610 nazorat) va 64 nafar o'qituvchi ishtirokida olib borildi. Tajriba konstatlovchi, shakllantiruvchi va nazorat bosqichlaridan o'tdi.




**9.** Tajriba-sinov natijalari shuni ko'rsatdiki, tajriba guruhida bilim sifati nazorat guruhiga nisbatan 30.1% yuqori bo'ldi (tajriba: 84.7 ball, nazorat: 65.1 ball). Farqning statistik ahamiyatliligi Styudent t-mezoni orqali tasdiqlandi (t = 32.45, p < 0.001). Tajriba guruhida 61.4% o'quvchilar yuqori darajaga erishdilar, nazorat guruhida esa 25.6%.

**10.** Bloom taksonomiyasining barcha darajalarida tajriba guruhi ustunlik qildi. Eng katta farq yuqori darajali kognitiv ko'nikmalarda (tahlil +24.5 ball, baholash +27.8 ball, yaratish +30.0 ball) kuzatildi. Bu axborot texnologiyalari asosida baholash nafaqat oddiy bilimlarni, balki murakkab fikrlash qobiliyatlarini ham rivojlantirishiga isbotdir.

**11.** So'rovnoma natijalari yuqori mamnuniyat darajasini ko'rsatdi: 82% o'quvchilar yangi tizimni qiziqarliroq deb hisoblaydilar, 91% o'qituvchilar vaqt tejashini ta'kidladilar, 81% ota-onalar yangi tizimni qo'llab-quvvatlaydilar. Gamifikatsiya elementlari (ball, daraja, nishonlar) o'quvchilar motivatsiyasini sezilarli darajada oshirdi.

**12.** Tajriba davomida ba'zi qiyinchiliklar (dastlab platformani o'rganish qiyinligi, ba'zi o'quvchilarda internet yo'qligi, texnik muammolar) aniqlandi, biroq ular hal qilinadigan xarakterga ega va umumiy ijobiy natijalarni inkor etmaydi.

## TAVSIYALAR

### Amaliyot uchun tavsiyalar

**1. Ta'lim muassasalari uchun:**
- EduAssess yoki shunga o'xshash axborot texnologiyalari asosida baholash platformalarini informatika fanidan bosqichma-bosqich joriy etish;
- Barcha o'qituvchilarni platformadan foydalanishga o'rgatish uchun maxsus treninglar tashkil etish;
- Zamonaviy kompyuter xonalari, tezkor internet aloqasi bilan ta'minlash;
- Ota-onalarni platformadan foydalanishga o'rgatish va ularga texnik yordam ko'rsatish.

**2. O'qituvchilar uchun:**
- Formativ baholashga ko'proq e'tibor qarting, nafaqat yakuniy natijani, balki o'rganish jarayonini ham baholang;
- Turli xil topshiriq turlari (test, amaliy mashq, loyiha, taqdimot) dan foydalaning;
- O'quvchilarga tezkor va konstruktiv qayta aloqa bering;
- Individual yondashuvni qo'llang, har bir o'quvchining rivojlanish dinamikasini kuzating;
- Gamifikatsiya elementlaridan o'rinli foydalaning, ammo haddan tashqari raqobatga yo'l qo'ymang.

**3. O'quvchilar uchun:**
- Yangi baholash tizimini qo'rqmasdan qabul qiling, xatolar o'rganishning tabiiy qismi;
- O'z natijalaringizni muntazam kuzating, rivojlanish dinamikangizni tahlil qiling;
- Zaif tomonlaringizni aniqlang va ularni yaxshilash uchun qo'shimcha mashg'ulotlar o'tkazing;
- O'qituvchi va ota-onalaringiz bilan natijalaringizni muhokama qiling;
- Platformaning barcha imkoniyatlaridan (elektron kutubxona, video darslar, forum) foydalaning.

**4. Davlat organlari uchun:**
- Respublika miqyosida axborot texnologiyalari asosida baholash tizimini joriy etish bo'yicha milliy dastur ishlab chiqish;
- Barcha maktablarni zamonaviy texnik vositalar bilan ta'minlash uchun moliyaviy resurslar ajratish;
- O'qituvchilarni qayta tayyorlash va malakasini oshirish tizimini takomillashtirish;
- O'zbek tilida milliy ta'lim platformalari (o'zbek EduAssess) yaratish va qo'llab-quvvatlash;
- Raqamli savodxonlik dasturlarini barcha ta'lim darajalarida amalga oshirish.

### Ilmiy tadqiqotlar uchun tavsiyalar

**1.** Axborot texnologiyalari asosida baholash tizimlarini boshqa fanlar (matematika, fizika, kimyo, tillar) uchun moslashtirib tadqiq etish.

**2.** Sun'iy intellekt va mashinali o'rganish texnologiyalarining pedagogik baholatishda qo'llanish imkoniyatlarini chuqurroq o'rganish.

**3.** O'quvchilarning psixologik holatiga (stress, tashvish) axborot texnologiyalari asosida baholashning ta'sirini tadqiq etish.

**4.** Uzoq muddatli ta'sir (longitudinal study) – bir guruh o'quvchilarni bir necha yil kuzatish va ularning davomli rivojlanishini tahlil qilish.

**5.** Blockchain texnologiyasi yordamida sertifikatlar va diplomlarni himoyalash, qalbakilashtirishning oldini olish yo'llarini tadqiq etish.

**6.** Virtual va kengaytirilgan reallik (VR/AR) texnologiyalarini bilimni baholashda qo'llash istiqbollarini o'rganish.

## 7-ILOVA. Statistik tahlil ma'lumotlari

### 7.1. T-mezon hisoblash (yakuniy bosqich)

**Berilganlar:**
- Tajriba guruhi: n₁ = 616, M₁ = 84.7, σ₁ = 9.2
- Nazorat guruhi: n₂ = 610, M₂ = 65.1, σ₂ = 11.8

**Hisoblash:**

1. Standart xato (Standard Error):
```
SE = √(σ₁²/n₁ + σ₂²/n₂)
SE = √(9.2²/616 + 11.8²/610)
SE = √(84.64/616 + 139.24/610)
SE = √(0.1374 + 0.2283)
SE = √0.3657
SE = 0.6047
```

2. T-qiymat:
```
t = (M₁ - M₂) / SE
t = (84.7 - 65.1) / 0.6047
t = 19.6 / 0.6047
t = 32.41
```

3. Erkinlik darajasi (degrees of freedom):
```
df = n₁ + n₂ - 2
df = 616 + 610 - 2
df = 1224
```

4. Kritik qiymat (α = 0.001):
- t_krit (df=1224, α=0.001) ≈ 3.29

5. Xulosa:
```
t_hisob (32.41) > t_krit (3.29)
p < 0.001
```

**Natija:** Tajriba va nazorat guruhlari o'rtasidagi farq statistik jihatdan juda ahamiyatli (highly significant).

### 7.2. Xi-kvadrat (χ²) testi - bilim darajalari taqsimoti

**Gipoteza:** Tajriba va nazorat guruhlarida bilim darajalari taqsimoti bir xilmi?

| Daraja | Tajriba (kuzatilgan) | Nazorat (kuzatilgan) | Jami |
|--------|---------------------|---------------------|------|
| Yuqori | 378 | 156 | 534 |
| Yaxshi | 186 | 238 | 424 |
| O'rta  | 48  | 168 | 216 |
| Past   | 4   | 48  | 52  |
| **Jami** | **616** | **610** | **1226** |

**Kutilgan qiymatlar** (agar farq bo'lmasa):

| Daraja | Tajriba (kutilgan) | Nazorat (kutilgan) |
|--------|-------------------|-------------------|
| Yuqori | 268.2 | 265.8 |
| Yaxshi | 213.0 | 211.0 |
| O'rta  | 108.5 | 107.5 |
| Past   | 26.1  | 25.9  |

**χ² hisoblash:**

```
χ² = Σ [(O - E)² / E]

Tajriba guruhi:
χ²₁ = (378-268.2)²/268.2 + (186-213.0)²/213.0 + (48-108.5)²/108.5 + (4-26.1)²/26.1
χ²₁ = 45.0 + 3.4 + 33.7 + 18.7 = 100.8

Nazorat guruhi:
χ²₂ = (156-265.8)²/265.8 + (238-211.0)²/211.0 + (168-107.5)²/107.5 + (48-25.9)²/25.9
χ²₂ = 45.4 + 3.5 + 34.0 + 18.9 = 101.8

Jami: χ² = 100.8 + 101.8 = 202.6
```

**Erkinlik darajasi:** df = (r-1)(c-1) = (4-1)(2-1) = 3

**Kritik qiymat:** χ²_krit (df=3, α=0.001) = 16.27

**Xulosa:** χ²_hisob (202.6) >> χ²_krit (16.27), p < 0.001. Farq statistik jihatdan juda ahamiyatli.
